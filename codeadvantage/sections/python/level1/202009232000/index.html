I. Input and Output basics

  1. Print a sentence of your choice with print().
  2. Ask the user to input a sentence and then repeat it back to the user with an exclamation point at the end.
  3. Use the special "\n" string to print a new line in the middle of a sentence. I won't tell you exactly how to do this -- it's up to you to experiment with it!
  4. Once you understand "\n", try "\t".

II. Personality Quiz

Expand on the small personality quiz we did in class. Collect answers to a bunch of questions and then list them out in whatever way you like. Then force your captive family members to take the quiz.

III. Mad Libs

Mad Libs is a game where one player asks the other for a series of parts of speech (e.g. "noun", "verb in past tense", "person's name") and uses those answers to fill in the blanks to a story that the other player doesn't see. Then, the story is read aloud and it should be completely absurd because the player providing the answers had no clue how the words would fit into the story.

Perhaps you can see how what we learned in class is perfectly suited to this kind of game.

  • Write your own story, or find one online, in the REPL editor as a string, leaving plenty of blanks for variables. Ask the user for input, prompting them like the player asking for parts of speech would. Store these answers and use them to fill in the blanks of the story.

    • It's important that the string comes after the answers are collected, or else the variables representing the answers won't be defined and the program will break!

    IV. Casting

    Note: decimal values are called floats in Python. Floats, just like ints, have a function for converting a value. For ints, this is int(), and appropriately for floats, this is float().

    There is also one for strings: str().

    1. Store various data types in different variables and try converting between them.
    2. Record what happens. Do any conversions cause errors? Look for strange edge cases.
    3. What happens when you convert a float to an int?

    V. Cashier

    Write a program called cashier.py that does the following:

    1. Ask user for an amount of money (the change owed) as input. This should be in the form "3.92" to represent 3 dollars and 92 cents. Hint: what data type is this? You will need to cast it to that data type, because input() always gives you a string.
    2. Convert this amount to pennies by multiplying by 100.
    3. Starting with dollars, then quarters, and so on to pennies, compute the fewest number of coins necessary to fill the given amount. For example, 3.92 cents is most efficiently broken up into 3 dollars, 3 quarters, 1 dime, 1 nickel, and 2 pennies.
    4. Print out how many of each coin is required to give the user their change.

    You will have to use a variable for each of the coins to store how many of them are needed.

    To make this work, you will need one more operator we didn't learn about in class: the modulus operator, written in Python as %. This is actually simpler than it seems; it just returns the remainder of the division of two numbers. For example, 9 % 2 = 1 and 10 % 5 = 0 and 92 % 25 = 17.

    Good luck!

      Fibonacci sequence

      In mathematics, there is something called the Fibonacci sequence, which is a special sequence of numbers defined as follows:


      F0 = 0
F1 = 1
Fn = Fn-1 + Fn-2

      Okay... but what the heck does that mean?!


      It's actually pretty simple. The Fibonacci sequence is just where you count by adding the two previous numbers in the list together to get the next one. For this to make sense, we need to start with two numbers already, so we say the first two numbers are 0 and 1.


      Let's see an example.


      First two numbers: 0, 1.
Next: 0 + 1 = 1.
Next: 1 + 1 = 2.
Next: 1 + 2 = 3.
Next: 2 + 3 = 5.
Next: 3 + 5 = 8.
Next: 5 + 8 = 13.
Next: 8 + 13 = 21.

      ...and so on. As you see, it actually makes a lot of sense! So the sequence so far can be written as:


      0,1,1,2,3,5,8,13,21,...

      Like many sequences, it goes on forever. It turns out that a lot of things in nature adhere to the Fibonacci sequence. For example, there's a famous thing called the Fibonacci spiral (pictured below) which you can find all throughout biology and art as it is very satisfying to look at.



      There's another kind of Fibonacci spiral, which instead of making squares out of the numbers of the sequence like above, counts the number of outward spirals of a sunflower plant, which happens to perfectly match the Fibonacci sequence.



      A similar idea appears in other plants. Here is an overlay of the Fibonacci sequence over a chamomile flower, demonstrating another one of these appearances in nature.



      For this exercise, the user will enter a number and you will then calculate and print that many numbers of the Fibonacci sequence.

      The Exercise

      Do the following in Python:

      1. Get an integer from the user and store it. Let's call this n
      2. Set two variables to represent the last two numbers in the Fibonacci sequence. They should start as 0 and 1.
      3. Use a while loop to repeatedly print the next number in the Fibonacci sequence until you've computed n Fibonacci numbers.
        • Consider: instead of adding 1 to a counter variable like we did in class, can you subtract 1 from n until you reach 0?

      Exercises

      Optional exercise if you enjoyed the Caesar cipher: make an encryption algorithm with three functions:

      1. gen(), which generates a completely random dictionary by which to associate letters with their encrypted versions
      2. encrypt(message, key), which encrypts it and takes as arguments a message and an encryption dictionary
      3. decrypt(message, key), which decrypts it.

      Keep in mind that you can't just pick the letters randomly, since you need to make sure that no two letters map to the same encrypted letter, or else you wouldn't be able to reliably reverse it! (Hint: shuffle the list of letters first and then associate them in order.)
